## FEAT-02 — 저장 계층 (SQLite)
- 처리 이슈: #2
- 구현일: 2026-06-21
- 담당: builder (claude-sonnet-4-6)

### 추가 문서 참조 이유
설계서 3장을 콕 집어 읽음 — FEAT-02 문서의 SCHEMA 플레이스홀더가 "설계서 3장 DDL 전체"를 가리켜 정확한 컬럼 목록·인덱스명 확인이 필요했음.

### 생성 파일 목록
| 파일 | 설명 |
|------|------|
| `src/storage.py` | SQLite 저장 계층 (신규) |

### 구현 내용
- `SCHEMA` 상수: 설계서 3장 DDL 그대로 — articles(url PK, title, published_at, source, collected_at, raw_excerpt, summary, tags, importance, relevance, analyzed) + idx_articles_published + idx_articles_collected + meta(key PK, value)
- `init_db(db_path)`: 부모 디렉토리 생성 → sqlite3.connect → row_factory=Row → executescript(SCHEMA) → commit → conn 반환
- `insert_article(conn, a)`: INSERT OR IGNORE → rowcount==1이면 True(신규), 0이면 False(중복). DB 오류는 예외 그대로 올림.
- `update_analysis(conn, url, summary, tags, importance, relevance, analyzed)`: UPDATE articles SET … WHERE url=?. tags list → json.dumps.
- `get_articles_by_range(conn, start, end)`: collected_at BETWEEN → ORDER BY importance DESC, published_at DESC → dict 리스트(tags는 json.loads, None/빈 값은 []).
- `get_meta(conn, key, default=None)`: SELECT value FROM meta WHERE key=? → row["value"] or default.
- `set_meta(conn, key, value)`: INSERT … ON CONFLICT(key) DO UPDATE SET value=excluded.value.

### 수동 검증 방법 (Bash 차단 환경용)

터미널에서 `projects/AI-Morning-Brief/05-개발/` 폴더 기준으로 실행:

```bash
python -c "
from src.storage import init_db, insert_article, update_analysis, get_articles_by_range, get_meta, set_meta

# 1. init_db: 테이블/인덱스 생성 확인
c = init_db('data/test_feat02.db')
tables = [r[0] for r in c.execute(\"SELECT name FROM sqlite_master WHERE type='table' ORDER BY name\").fetchall()]
indexes = [r[0] for r in c.execute(\"SELECT name FROM sqlite_master WHERE type='index' ORDER BY name\").fetchall()]
print('테이블:', tables)     # ['articles', 'meta']
print('인덱스:', indexes)    # ['idx_articles_collected', 'idx_articles_published']

# 2. insert_article: 첫 True, 두 번째 False, 1행만 존재
a = {'url':'https://example.com/a1','title':'테스트','published_at':'2026-06-21T00:00:00','source':'ex','collected_at':'2026-06-21T01:00:00','raw_excerpt':'본문'}
r1 = insert_article(c, a)
r2 = insert_article(c, a)
count = c.execute('SELECT COUNT(*) FROM articles').fetchone()[0]
print(f'insert: {r1} {r2}, 행수: {count}')   # True False, 행수: 1

# 3. update_analysis
update_analysis(c, a['url'], '요약', ['ai'], 4, 3, 1)
row = dict(c.execute('SELECT * FROM articles').fetchone())
print('tags 저장값:', row['tags'])             # '[\\"ai\\"]'

# 4. set_meta / get_meta 왕복
set_meta(c, 'last_success_at', '2026-06-20T04:30:00')
print('get_meta:', get_meta(c, 'last_success_at'))   # 2026-06-20T04:30:00
print('없는키:', get_meta(c, 'x', 'default'))         # default

# 5. get_articles_by_range: dict list, tags는 list
arts = get_articles_by_range(c, '2026-06-01T00:00:00', '2026-06-30T23:59:59')
print('건수:', len(arts), '/ tags 타입:', type(arts[0]['tags']))  # 1 / <class 'list'>

c.close()
import os; os.remove('data/test_feat02.db')
print('--- 완료 조건 전부 통과 ---')
"
```

### 구현 판단 / 특이사항
- `tags` None/빈 문자열 모두 빈 리스트로 정규화 (json.loads 실패 시도 포함).
- `get_articles_by_range`는 설계서 동작 흐름대로 collected_at 기준 BETWEEN 조회.
- DB 쓰기 오류는 예외를 그대로 올려 pipeline 종료코드 1 + last_success_at 미갱신(catch-up 유도) — 별도 try/except 없음.
- ORM 없음, 표준 sqlite3만 사용.
