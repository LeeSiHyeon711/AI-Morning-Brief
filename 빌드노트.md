### 수동 검증 방법 (Bash 차단 환경용)

터미널에서 `projects/AI-Morning-Brief/05-개발/` 폴더 기준으로 실행:

```bash
# 1. 의존성 설치
pip install -r requirements.txt

# 2. --help: 7개 옵션 출력 확인
python main.py --help

# 3. --test: ImportError 없이 종료코드 0 확인
python main.py --test
echo "종료코드: $?"

# 4. load_config() + env override 확인
python -c "
from src.config_loader import load_config
cfg = load_config()
print('type:', type(cfg))
print('paths:', cfg.get('paths'))
"

MORNINGBRIEF_DB=/tmp/x.db python -c "
from src.config_loader import load_config
cfg = load_config()
print(cfg['paths']['db'])  # 기대: /tmp/x.db
"

# 5. load_sources() — sources.yaml 없음 → example fallback + WARNING
python -c "
import logging; logging.basicConfig(level=logging.WARNING)
from src.config_loader import load_sources
srcs = load_sources()
print('소스 수:', len(srcs))
print('첫 소스:', srcs[0] if srcs else '없음')
"

# 6. load_operator_profile() — 파일 있을 때 is_default=False
python -c "
from src.config_loader import load_operator_profile
text, is_default = load_operator_profile()
print('is_default:', is_default)
"

# 7. operator_profile 부재 시 DEFAULT_PROFILE + WARNING
python -c "
import logging; logging.basicConfig(level=logging.WARNING)
from src.config_loader import load_operator_profile
text, is_default = load_operator_profile('nonexistent.md')
print('is_default:', is_default)  # 기대: True
"

# 8. load_secrets()
python -c "
from src.config_loader import load_secrets
s = load_secrets()
print('keys:', list(s.keys()))
"
```

### 구현 판단 / 특이사항
- `load_sources`의 `example_path` 계산 시 `os.path.dirname(path)`를 사용해 사용자 지정 path에서도 올바른 경로를 참조하도록 구현 (FEAT 문서의 하드코딩 방식 대비 안정성 개선, 동작은 동일)
- `main.py`의 `sys.exit(main())` 패턴으로 종료코드 0/비0을 명시적으로 처리
