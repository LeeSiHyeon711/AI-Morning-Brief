## Claude / Claude Code 관련 소식

- [v2.1.204](https://github.com/anthropics/claude-code/releases/tag/v2.1.204) — Claude Code v2.1.204 패치 릴리즈. 헤드리스 세션의 SessionStart 훅에서 훅 이벤트가 스트리밍되지 않아 원격 워커가 유휴 상태로 회수되던 버그가 수정되었다. CI/CD 파이프라인이나 자동화 환경에서 Claude Code를 운영하는 경우 즉시 업데이트가 권장된다.

- [Why the rise of open source AI isn’t hurting Anthropic … yet](https://techcrunch.com/2026/07/07/why-the-rise-of-open-source-ai-isnt-hurting-anthropic-yet/) — 오픈소스 AI 모델의 급성장이 Anthropic 등 프론티어 랩에 직접적 타격을 주지 않는 이유를 분석한 기사. 오픈소스와 프론티어 모델이 동일 생애주기의 서로 다른 단계를 각각 포착하며 공존한다는 시각이 제시됐다. AI 시장 구조에 대한 전략적 이해에 유용한 관점을 제공한다.

## MCP / Agent / 자동화 관련 소식

- [v2.1.203](https://github.com/anthropics/claude-code/releases/tag/v2.1.203) — Claude Code v2.1.203에서 로그인 만료 경고, 수동 권한 모드 UI 표시, MCP roots/list에 추가 작업 디렉토리 반영 등 다수 UX 개선이 포함되었다. macOS에서 백그라운드 에이전트 세션 전환 시 15~20초 지연 버그 및 데몬 토큰 만료 시 세션 불응 문제도 함께 수정되었다. 멀티 에이전트·백그라운드 세션 환경의 안정성이 크게 향상됐다.

## 상상공방에 적용할 수 있는 아이디어

- Claude Code 백그라운드 에이전트 개선(v2.1.203)을 활용해 상상공방 내 자동화 파이프라인(코드 리뷰·테스트 자동화 등)에서 헤드리스 세션 기반 워크플로우 구성을 검토할 시점. SessionStart 훅 버그(v2.1.204)가 수정됐으므로 CI 환경 적용에 신뢰성 확보.
- GitHub Copilot Business에서 Kimi K2.7이 지원되므로, GPT-4o 대비 Kimi K2.7의 한국어 코딩 보조 성능을 상상공방 개발 업무에서 직접 비교 테스트해볼 기회.
- 오픈소스 AI가 프론티어 모델과 경쟁하는 게 아니라 '초기 탐색(오픈소스) → 프로덕션(프론티어)'의 파이프라인을 형성한다는 분석은, 상상공방의 AI 도입 전략 설계 시 '개발·프로토타입에는 오픈소스, 서비스 배포에는 Claude/GPT'라는 이중 전략의 근거로 활용 가능.
- 마이크로소프트의 자체 모델 전환 트렌드는 외부 API 비용 의존 리스크를 시사. 상상공방 서비스의 AI API 비용 구조를 점검하고, 로컬 or 소형 모델과 프론티어 모델의 혼용(cost-tier routing) 전략 수립을 고려.
- Vercel eve의 Chat SDK 채널 통합으로 WhatsApp·Slack 등 멀티채널 AI 에이전트를 단일 코드베이스로 배포 가능. 상상공방 고객 대응 또는 내부 업무용 챗봇 통합에 저비용으로 적용 가능한 아키텍처.
